How to Use

1
Go to your OIDC App page in Okta Admin, click the bookmarklet. The floating panel appears with Client ID and 5 derived URLs auto-filled.
2
Navigate to the Custom Auth Server page, click again. Server ID is extracted and 3 more URLs are derived. Panel updates.
3
For manual values: highlight text on the page (Client Secret, JWK, Key ID), then click the bookmarklet. A dropdown appears — pick the field to assign it to.
4
Click "Copy Config JSON" in the panel, go to o4aa-demo.vercel.app/settings, click "Paste Config JSON". Done.
How it works: The bookmarklet runs entirely in the current page's JavaScript context — no external scripts loaded, no network requests, no CSP issues. Values accumulate in sessionStorage as you navigate between Okta admin pages (cleared when you close the tab). The panel is draggable, shows field-level status, and copies the complete config JSON to your clipboard.
